# Momentum 项目 Prompt 题集

> 适用于 Momentum A 股量化分析与交易系统（FastAPI + React + PostgreSQL + Redis）。
> 题号前缀含义：`F` = Feature 迭代，`N` = 0-1 模块构建，`T` = 代码测试，`R` = 代码重构，`E` = 工程化。

---

## 一、Feature 迭代（共 20 题）

### F-01 【Feature 迭代】形态识别新增成交量过滤与K线标注
当前 `Patterns` 页面只根据收盘价识别经典 K 线形态（头肩顶、双重底等），结果只展示触发日期和估计胜率。我希望在现有的形态识别流程基础上增加一层"量价确认"：在原有形态判断之上叠加"近 N 日成交量相对均量放大倍数 ≥ 阈值"的过滤条件，N 与阈值都在前端面板上以滑块或输入框可调，并需要持久化到 `ScreeningPreset` 或类似存储里以便下次直接复用。识别结果列表里要新增一个"量能确认强度"列（计算最近一根触发 K 线的成交量与过去 N 日均量的比值），并按这个强度做二次排序。识别结果点击后能弹出一个迷你 K 线图，把触发当天的位置在图上用三角形标注出来，同时高亮显示量能放大的那几根 K 线。最后要在前端能完整走通：选形态 → 调阈值 → 扫描 → 查看带标注的迷你 K 线 → 保存当前阈值方案。

### F-02 【Feature 迭代】Dashboard 新增涨跌幅榜单与自动刷新
当前 `Dashboard` 只有四张统计卡片和一张市值分布饼图。请在不破坏现有视觉风格的前提下，扩展出一个"今日涨跌幅排行榜"模块：左右两栏分别展示涨幅前 10 和跌幅前 10，每行显示股票名称、代码、最新价、涨跌幅、所属市场（SH/SZ），涨跌幅要用红涨绿跌的色块表示。后端需要在 `/dashboard` 下新增对应接口，从 `StockSnapshot` + `DailyPrice` 计算最近一个交易日的涨跌幅。前端要支持三种刷新模式：手动刷新按钮、30 秒自动轮询、关闭轮询，并提供下拉切换。每行支持点击跳转到 `Visual` 页并自动填入该股票和最近 60 天日期范围。要保证页面打开时即使数据为空也有合理的空态展示，并且自动刷新在切换路由后能正确清理。

### F-03 【Feature 迭代】回测页面新增多策略对比与基准曲线
`Backtest` 页面当前只支持单一策略多股票回测，权益曲线把多只股票画在一起。请扩展为"多策略对比模式"：用户可以一次选择 2~4 个策略、相同的股票列表和日期范围，运行后在同一张图上画出每个策略的等权组合净值曲线。同时引入"基准曲线"，可选沪深 300 / 上证指数（如果没有真实数据可以用所选股票池的等权买入持有作为替代基准），与各策略曲线一起展示。指标卡片要改造成矩阵布局：行是策略，列是年化收益、最大回撤、夏普、胜率、相对基准超额。后端要修改 `/backtest/run` 接口或新增 `/backtest/compare` 来支持批量策略执行，并在前端做并发控制（防止用户重复点击）。结果支持导出为 CSV，并保留最近 5 次对比记录在 localStorage 里以便快速切换查看。

### F-04 【Feature 迭代】选股方案的分组管理与历史快照
`Screening` 现有的方案保存只是一个扁平的名字列表，方案多了不好找。请扩展为"分组 + 历史快照"两层结构：方案可以归属于用户自建的分组（例如"价值股"、"成长股"），分组本身可以重命名、删除、拖拽排序。每次基于某个方案运行筛选时，要把当时的入参、命中股票数、命中股票列表前 50 条快照写入一张新表 `ScreeningHistory`，并在方案详情里能看到"最近 10 次运行结果"的时间轴，点击某一次能快速恢复当时的入参并重新筛选。前端要在方案侧栏支持折叠分组、搜索方案名，并要做删除二次确认。后端要相应地补充分组的 CRUD 接口和历史快照接口，注意分组删除时方案的归属处理（移到"未分组"而不是级联删除）。

### F-05 【Feature 迭代】K线副图扩展 BOLL/KDJ/MACD 指标
`Visual` 当前只有 MA5/MA20/成交量三个简单叠加。请扩展为标准的"主图 + 副图"专业形态：主图保留 K 线与均线、新增 BOLL 布林带（上轨/中轨/下轨）作为可勾选叠加；副图支持最多同时显示两个指标，可选项包括 MACD（柱状图 + DIF/DEA 双线）、KDJ（K/D/J 三线）、RSI（单线 + 30/70 参考横线）、成交量。副图与主图必须在 ECharts 中正确联动（缩放、tooltip 十字光标跨图同步）。所有指标的参数（如 MACD 的 12/26/9，BOLL 的 20/2）需要在前端有"参数面板"可调，调整后图表实时更新。后端可以直接复用 `indicators.py` 的算法，也可以让前端基于行情自行计算，但要确保和后端结果一致。最后要保证在切换股票或日期时副图状态被正确刷新。

### F-06 【Feature 迭代】同步任务支持取消与失败重试
当前 `DataCenter` 的后台同步只能等其自然结束或报错。请在现有 `SYNC_STATE` + `BackgroundTasks` 基础上扩展任务控制能力：增加"取消正在执行的同步"接口，前端面板上对应"终止"按钮（带二次确认）；同步过程中出现单只股票失败时不要整体崩溃，要写入失败列表并继续，结束后在前端展示"成功 X 只 / 失败 Y 只"，并提供"仅重试失败项"按钮，点击后只对失败列表里的股票重新发起同步。需要把失败明细持久化到 `DataSyncLog` 里方便后续追查，同时前端进度条需要能区分"运行中 / 已取消 / 部分失败 / 完成"四种状态，对应不同颜色与图标。注意取消机制要避免在数据库写入中间留下脏数据，至少要保证单只股票级别的原子性。

### F-07 【Feature 迭代】系统日志多维筛选与导出
`SystemLogs` 当前只有简单的分页和按日期范围删除。请扩展为更实用的运维视图：顶部增加多维筛选条件（数据源下拉、同步类型下拉、状态下拉 success/failed、关键字模糊匹配 message 字段、时间区间），所有条件可以叠加，并要在 URL query 中保存以便分享和刷新后保留。日志表格要支持按时间排序切换、每行展开看完整 message（部分 message 可能很长），并加入"按数据源 / 按状态"的分组聚合视图（小卡片显示数量分布）。同时新增一个"导出当前筛选结果"按钮，支持 CSV 和 Excel 两种格式，导出内容必须严格遵循当前筛选条件而不是导出全部。后端要相应支持多条件查询和导出接口，注意大数据量下的分页与流式响应。

### F-08 【Feature 迭代】策略参数模板与收藏
`Strategies` 当前的参数都是写死的"短/长周期"两个通用字段。请扩展为"策略参数模板"系统：每个策略可以有多组参数模板（例如 MA 交叉策略下保存"5/20 经典版"、"10/30 慢速版"），模板支持命名、描述、保存、加载、删除、设为默认。前端在策略卡片上要显示该策略已保存的模板数量与默认模板名称，点击"配置回测"时下拉选择模板自动填充参数。后端新增 `StrategyParamPreset` 模型与对应 CRUD 接口。再增加"收藏策略"功能：用户可以把常用策略标记为收藏（用户级别），策略库页面顶部增加"全部 / 我的收藏"切换 Tab。最后要做一个小细节：参数模板支持复制（基于现有模板创建新模板时预填字段），方便快速派生变种。

### F-09 【Feature 迭代】回测交易明细与买卖点K线标注
当前回测只输出年化、夏普等汇总指标和权益曲线，黑盒感比较强。请扩展为可追溯的"交易明细模式"：回测引擎在生成 signal 后，根据 signal 由 0→1 视为开仓买入、1→0 视为平仓卖出，记录每笔交易的开仓日、开仓价、平仓日、平仓价、持仓天数、单笔收益率。后端返回结果中新增 `trades` 字段。前端在回测结果区下方新增"交易明细"折叠面板，以表格形式展示所有交易，并支持按收益率排序、按持仓天数筛选。点击某条交易，权益曲线旁的小型 K 线图（新增一张）会跳转到对应日期附近，并在 K 线上用上下三角箭头标注买卖点。所有交互需要在选择不同股票时正确切换。整体要在前端完整跑通：选策略 → 跑回测 → 看交易明细 → 点击单笔 → K 线上标注。

### F-10 【Feature 迭代】筛选自定义条件支持表达式
`screen_stocks` 当前的自定义筛选只支持"字段 + min/max 范围"。请扩展为支持简单的字段表达式：用户可以在前端输入诸如 `pe_ratio / pb_ratio < 5`、`market_cap > 1e10 AND rsi < 70`、`(close - ma20) / ma20 > 0.05` 这样的表达式。需要设计一个白名单 + 解析器（不能直接 eval 用户输入），允许的字段就是当前 `allowed_fields`，操作符限制在 `+ - * /  > >= < <= = AND OR ()`。后端实现一个安全的表达式解析模块（例如基于 `ast` 模块的受限解析，或者引入轻量级表达式库），把表达式翻译为对应的 SQL WHERE。前端要在"自定义条件"区域增加表达式输入框，提供字段提示和实时语法校验（错误时输入框红框提示），支持点击"应用"运行筛选。最终在前端完整跑通至少 3 个示例表达式。

### F-11 【Feature 迭代】K线对比模式与归一化
`Visual` 当前只能看单只股票。请扩展为可选的"对比模式"：在股票选择器旁增加"+ 添加对比"按钮，允许同时叠加最多 4 只股票的走势在同一张图上。由于不同股票价格差异巨大，必须支持"归一化"开关：开启时把每只股票的起始日价格归一为 100，后续按相对涨跌幅计算，关闭时显示原始价格（不同 y 轴或自动缩放）。每只对比股票要有独立颜色和图例，可单独从图上隐藏。对比模式下原来的副图（成交量）需要变成"对比汇总条"显示每只股票当日区间累计涨跌幅。最终要前端可以验证：先选 A 股 → 加入对比 B 股 → 加入对比 C 股 → 切归一化开关 → 隐藏 B 股 → 切换日期范围，整个过程数据正确联动且不闪烁。

### F-12 【Feature 迭代】形态识别结果一键回测
现有形态识别产生一组 (股票, 形态, 触发日期) 的结果，但用户没有手段验证"如果当时按这个形态买入持有 N 天会怎样"。请在 `Patterns` 页面增加"基于触发点回测"功能：在结果列表的每条结果上增加"快速回测"按钮，点击后弹出一个迷你配置弹窗（持有期 N 天、止盈百分比、止损百分比），确定后调用回测引擎计算从触发日开始持有 N 天（或触发止盈止损）的收益，结果就地展示（小卡片显示收益、最大浮亏、是否触发止损）。同时新增一个"批量回测当前结果"按钮，把列表中所有触发点统一回测后输出一个汇总统计：胜率、平均收益、平均持仓天数、最佳/最差案例。后端可以新增一个 `/patterns/replay` 接口承载这部分逻辑。整体在前端要能完整闭环：扫形态 → 单点验证 → 批量统计。

### F-13 【Feature 迭代】Dashboard 自定义关注股票卡片
当前 Dashboard 的市值分布饼图固定从全市场 top6 来，对实际操作没什么用。请新增"我的关注"模块：用户可以从顶部"+ 添加关注"按钮里搜索并加入股票，关注列表持久化到后端（新增表 `UserWatchItem` 或类似），每只关注股票以一张小卡片展示最新价、当日涨跌幅、5 日累计涨跌幅、近 30 日迷你折线图（mini sparkline，可用 ECharts）。卡片支持拖拽排序与删除（带二次确认）。新增分组维度：用户可以把关注股票放到自建分组里（例如"白马"、"科技"），顶部用 Tab 切换分组只看该组。后端要提供分组与关注项的 CRUD 接口，并要鉴权（不同用户的关注互相隔离）。前端要做空态、加载态、并发添加去重。最终需要在前端走通完整流程：添加 → 分组 → 排序 → 看 mini chart → 删除。

### F-14 【Feature 迭代】数据中心单股快速同步与缺口检测
现有数据中心只能"全市场同步"或"按日期同步全部"，没法针对单只股票。请扩展：在"股票清单预览"列表里每一行加一个"快速同步"按钮，点击后弹窗让用户选择该单只股票的同步日期范围，确定后只发起这一只股票的日线同步任务（复用 `sync_daily` 但传入单元素 symbols 列表），并要复用现有的进度面板。同时增加"缺口检测"功能：在数据中心新增一个"数据完整性"页签，调用 `/data/integrity` 对全市场或用户选定子集做检测，把每只股票的实际数据起止日期、缺失天数、重复天数列在一张可排序的表格中，缺失天数大于 0 的行高亮显示；表格每行支持"一键补齐"按钮，自动以缺失的最大连续区间作为同步范围发起同步。前端要能跑通：检测 → 看到缺口 → 一键补齐 → 进度更新 → 再检测验证。

### F-15 【Feature 迭代】系统日志的任务统计概览
`SystemLogs` 目前只有列表视图，对于运维同学想看"总体趋势"很不友好。请在系统日志页顶部新增一个"统计概览"卡片区：用 ECharts 画三个可视化图：1）最近 7 天每日成功/失败任务数的堆叠柱状图；2）按数据源统计的成功率横向条形图（AKShare、东方财富、新浪等）；3）按同步类型（stock_list、daily、incremental、full、scheduled）的任务数饼图。统计时间范围支持下拉切换（7 天、30 天、90 天）。同时增加"耗时分析"：如果 `DataSyncLog` 没有 duration 字段，请扩展模型加上 `duration_ms`，并修改日志写入处补全该字段，再画一个"任务平均耗时趋势折线图"。整体面板要支持折叠以节省屏幕空间。前端要能完整看到所有图表正确加载，并在切换时间范围时联动刷新。

### F-16 【Feature 迭代】批量回测汇总报告与排序筛选
当前 `Backtest` 跑多只股票时每只股票的结果只是平铺在右侧。请新增一个"汇总报告"视图（Tab 切换）：以表格形式展示所有股票的回测结果（年化、最大回撤、夏普、胜率、profit_factor），支持每列排序，支持按指标范围筛选（例如"夏普 > 1"、"最大回撤 > -20%"）。表格底部要显示当前筛选下的统计汇总：平均年化、平均夏普、胜率中位数、表现最好/最差的股票。新增"导出当前筛选汇总"按钮，导出 CSV/Excel。表格行支持点击展开看单只的权益曲线缩略图。后端如果性能不够要考虑批量并发回测（用 `concurrent.futures` 或 asyncio），同时要在前端展示批量回测进度（已完成 X/Y）。完成后要在前端走通：选 10+ 只股票 → 跑回测 → 切到汇总视图 → 按夏普排序 → 筛选 → 导出。

### F-17 【Feature 迭代】K线画线工具
专业 K 线分析离不开画线。请在 `Visual` 图上新增工具栏：水平线、趋势线（两点确定直线）、矩形框、文字标注，四种工具。点击工具进入对应绘制模式，鼠标在图上点击两点完成绘制；绘制完成后可点选已绘制的图形进行修改坐标或删除。所有画线结果按"股票 + 周期"维度持久化到后端（新增 `ChartAnnotation` 模型），切换股票时只显示属于当前股票的标注。前端要解决 ECharts 上自定义画线的坐标转换问题（像素坐标 ↔ K 线时间/价格坐标），并保证在数据缩放（zoom）时画线跟随移动而不是漂浮在屏幕上。最终前端要能验证：画一条趋势线 → 缩放 → 切换股票 → 切回 → 标注仍在 → 删除某条标注 → 持久化生效。

### F-18 【Feature 迭代】导出字段自定义与模板保存
现有 `/export` 导出的字段是写死的（直接 `DataFrame.to_csv`）。请改造为"字段可选 + 模板保存"：导出弹窗里新增字段勾选区，列出所有可导出字段（trade_date、open、high、low、close、volume、amount、symbol、name、industry、market_cap、pe_ratio、pb_ratio 等），并按"基础信息 / 行情 / 估值 / 因子"分组折叠展示。用户勾选完之后可以"保存为模板"，模板名可自定义；下次打开导出弹窗顶部下拉里可以直接选择已保存模板，自动勾上对应字段。导出时后端按字段勾选动态决定返回哪些列。模板要持久化到后端（新增 `ExportTemplate` 模型），并支持删除、重命名。前端要完整走通：勾选 → 保存模板 A → 关闭弹窗 → 再次打开 → 选模板 A → 字段被勾选 → 导出 → 文件只含勾选字段。

### F-19 【Feature 迭代】筛选结果一键加入回测股票池

### N-16 【0-1 模块】投资组合管理：在个体下单视角之上提供结构化的"组合视角"。后端引入 `Portfolio`（组合：用户、名称、描述、基准代码、再平衡频率）与 `PortfolioHolding`（组合、股票、目标权重，校验同组合权重总和为 100%）；接口涵盖组合 CRUD、组合内持仓 CRUD（含权重批量保存接口校验总和约束）、组合历史净值回算接口（指定起始日按目标权重定期再平衡返回组合净值时间序列与对比基准曲线）、组合关键指标接口（年化、最大回撤、夏普、信息比率、与基准的相关性）、再平衡建议接口（当某成分股最新权重偏离目标超过阈值时给出买入或卖出量建议）。前端 `/portfolio_mgmt` 页面：左侧组合列表（增 / 复制 / 删除），右侧组合详情——权重饼图、成分股表格（含目标权重、当前权重、偏离、操作建议）、历史净值曲线（叠加基准曲线）、关键指标卡片矩阵、再平衡建议高亮提示区。

### N-17 【0-1 模块】风险指标计算：项目长期缺乏专业的风险度量视角。后端 `app/services/risk.py` 提供：单只或多只股票的 VaR 接口（历史模拟法与参数法两种、95% 与 99% 两种置信度、支持指定持有期）、相对基准的 Beta 与 Alpha 接口（在指定日期范围内对所选基准做线性回归返回 Beta、Alpha、R²、p-value）、相关性矩阵接口（多只股票/指数两两皮尔逊相关系数）、年化波动率接口、最大回撤接口、滚动 Beta 时间序列接口。前端 `/risk` 页面顶部参数配置区（股票多选、日期范围、基准代码下拉、置信度选择、持有期），下方四个子区域——VaR 卡片（多置信度对比展示）、Beta 表格（含拟合优度与显著性）、相关性热力图（ECharts heatmap）、风险指标对比表（按股票排序）；具备数据不足导致回归失败时的友好兜底提示。

### N-18 【0-1 模块】智能选股推荐：在硬规则筛选之外提供"综合打分排序"能力。后端 `app/services/recommend.py` 定义可插拔的打分规则集合（每条规则含权重与评分函数：PE 在 10~20 给满分超出按距离衰减、PB 在 1~3 给满分、RSI 在 30~50 给满分、最近 20 日动量大于 0 给满分、MACD 金叉给满分、行业排名前 30% 给满分、市值在 100~1000 亿给满分等）；接口涵盖默认评分卡的 Top-N 推荐、自定义权重的实时评分（请求体传入各规则权重数组）、评分明细查询（指定股票返回每条规则贡献的得分）、评分卡方案保存与加载。前端 `/recommend` 页面：左侧权重配置区（每条规则一行：启停开关 + 权重滑块 + 规则说明 + 当前默认值），右上推荐结果表格（按总分排序，含总分、各规则得分小色条），右下方明细面板（hover 任意股票展开该股票每条规则贡献度），顶部"保存当前评分卡"按钮持久化方案、下拉切换历史方案。
